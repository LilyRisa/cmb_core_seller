# Automation Flow Builder — S2: Postback + Interactive (button) messages

> **For agentic workers:** implement task-by-task, commit after each task (per project preference "commit sau mỗi lần làm"). Steps use `- [ ]` for tracking.

**Goal:** Close the missing half of the Facebook automation loop: send **interactive button messages** and react to **button presses (postback)** to advance a flow. After S2 a flow can: send a message with buttons → pause → when the buyer taps a button, resume down the edge for that button. No UI yet (S3/S4).

**Builds on:** S1 (`FlowEngine`, `FlowGraph`, `NodeExecutorRegistry`, `flow_runs`, `automation_flows`). Spec: `docs/superpowers/specs/2026-05-28-facebook-automation-flow-builder-design.md` §6, §7, §13.

**Architecture (unchanged invariants):**
- **Core never names a provider.** New capabilities `outbound.interactive` + `inbound.postback` on the connector; the node executor checks `supports()` before sending; unsupported → node fails + logs (no spam, no exception breaking the run).
- **All outbound goes through `OutboundMessageService` → `SendMessage` job → connector**, so audit + 24h `OutboundWindowGuard` (reactive: FB rejects → `OutboundWindowClosed`) behave identically for interactive sends.
- **Adding the new node type = one executor class + one `register()` line** (no engine edit), same as S1.
- A button press is modeled as "the reply that resumes the waiting `send_buttons` node": the postback payload encodes `{node_id, handle}`; the engine resumes by following the edge whose `sourceHandle === handle`.

**Tech:** Laravel 11, PHP 8.3, PHPUnit. Test DB = SQLite `:memory:`, `QUEUE_CONNECTION=sync`. All commands from `app/`. No hardcoded provider names in `Flows`; no sample data in shipped code.

## Node graph addition (locked; S3/S4 canvas emits exactly this)

```jsonc
{
  "nodes": [
    { "id": "t",   "type": "trigger",      "data": {} },
    { "id": "ask", "type": "send_buttons", "data": {
        "text": "Bạn cần gì ạ?",
        "buttons": [
          { "id": "b_buy",  "label": "Mua hàng",      "type": "postback" },
          { "id": "b_ship", "label": "Phí ship",      "type": "postback" },
          { "id": "b_web",  "label": "Xem website",   "type": "url", "url": "https://shop.vn" }
        ]
    }},
    { "id": "buy",  "type": "send_message", "data": { "text": "Cảm ơn, mời bạn đặt hàng…" } },
    { "id": "ship", "type": "send_message", "data": { "text": "Phí ship 30k toàn quốc." } },
    { "id": "end",  "type": "end", "data": {} }
  ],
  "edges": [
    { "id": "e0", "source": "t",   "target": "ask",  "sourceHandle": null },
    { "id": "e1", "source": "ask", "target": "buy",  "sourceHandle": "b_buy" },
    { "id": "e2", "source": "ask", "target": "ship", "sourceHandle": "b_ship" },
    { "id": "e3", "source": "ask", "target": "end",  "sourceHandle": null }
  ]
}
```

- `send_buttons` **sends** the interactive message then **waits** (its own id becomes `current_node_id`).
- Each `postback` button's wire payload is generated by the executor as `FlowPostbackPayload::encode(node_id="ask", handle=button.id)`. The matching outgoing edge uses `sourceHandle = button.id`.
- A `null`-handle edge from `ask` (here `e3`) is the **fallback** taken if the buyer types text instead of tapping (S1 text-resume path) or taps an unmapped button.
- `url` buttons open a web page (no postback, no edge needed).

## Task 4 — Engine side: payload codec, queueInteractive, SendMessage branch, resume handle

**Files:**
- Create `app/app/Modules/Messaging/Services/Flows/FlowPostbackPayload.php`:
  ```php
  <?php
  namespace CMBcoreSeller\Modules\Messaging\Services\Flows;
  /**
   * Mã hoá/giải mã payload postback của nút bấm flow. Builder sinh payload (người
   * dùng không bao giờ thấy/nhập). Định dạng JSON có marker để phân biệt postback
   * flow với postback khác (get_started, menu…). FB payload tối đa 1000 ký tự.
   */
  final class FlowPostbackPayload
  {
      private const MARKER = 'flow';
      public static function encode(string $nodeId, string $handle): string
      {
          return (string) json_encode(['t' => self::MARKER, 'n' => $nodeId, 'h' => $handle]);
      }
      /** @return array{node_id:string, handle:string}|null */
      public static function decode(string $payload): ?array
      {
          $data = json_decode($payload, true);
          if (! is_array($data) || ($data['t'] ?? null) !== self::MARKER) { return null; }
          $node = (string) ($data['n'] ?? ''); $handle = (string) ($data['h'] ?? '');
          if ($node === '' || $handle === '') { return null; }
          return ['node_id' => $node, 'handle' => $handle];
      }
  }
  ```
- Modify `app/app/Modules/Messaging/Services/OutboundMessageService.php` — add:

## Task 5 — `send_buttons` executor + `AdvanceFlowOnPostback` listener + wiring

**Files (create):**
- `app/app/Modules/Messaging/Services/Flows/Nodes/SendInteractiveNodeExecutor.php` (type `send_buttons`):
  ```php
  <?php
  namespace CMBcoreSeller\Modules\Messaging\Services\Flows\Nodes;
  use CMBcoreSeller\Integrations\Messaging\MessagingRegistry;
  use CMBcoreSeller\Modules\Messaging\Services\Flows\FlowContext;
  use CMBcoreSeller\Modules\Messaging\Services\Flows\FlowPostbackPayload;
  use CMBcoreSeller\Modules\Messaging\Services\Flows\Graph\FlowNode;
  use CMBcoreSeller\Modules\Messaging\Services\OutboundMessageService;
  use Illuminate\Support\Facades\Log;
  /**
   * Gửi tin có nút bấm rồi CHỜ buyer bấm (postback). Mỗi nút postback được gán
   * payload mã hoá {node_id, handle=button.id}; edge ra dùng sourceHandle=button.id.
   * Kiểm capability provider trước — không hỗ trợ ⇒ node fail + log (không vỡ luồng).
   */
  class SendInteractiveNodeExecutor implements NodeExecutor
  {
      public function __construct(private OutboundMessageService $outbound, private MessagingRegistry $registry) {}
      public function type(): string { return 'send_buttons'; }
      public function execute(FlowNode $node, FlowContext $ctx): NodeResult
      {
          $sent = (array) ($ctx->run->context['_sent'] ?? []);
          if (in_array($node->id, $sent, true)) { return NodeResult::wait(); } // đã gửi ⇒ tiếp tục chờ

          $provider = $ctx->conversation->provider;
          if (! $this->registry->has($provider) || ! $this->registry->for($provider)->supports('outbound.interactive')) {
              Log::warning('flow.interactive_unsupported', ['provider' => $provider, 'flow_run' => $ctx->run->id]);
              return NodeResult::fail('interactive_unsupported');
          }

          $text = trim((string) ($node->data['text'] ?? ''));
          $buttons = [];
          foreach ((array) ($node->data['buttons'] ?? []) as $b) {
              $b = (array) $b;
              $label = (string) ($b['label'] ?? $b['title'] ?? '');
              if ($label === '') { continue; }
              $type = ((string) ($b['type'] ?? 'postback')) === 'url' && ! empty($b['url']) ? 'url' : 'postback';
              $buttons[] = $type === 'url'
                  ? ['type' => 'url', 'title' => $label, 'url' => (string) $b['url']]
                  : ['type' => 'postback', 'title' => $label, 'payload' => FlowPostbackPayload::encode($node->id, (string) ($b['id'] ?? $label))];
          }
          if ($text === '' && $buttons === []) { return NodeResult::advance(); } // node rỗng ⇒ bỏ qua

          $this->outbound->queueInteractive($ctx->conversation, ['text' => $text, 'buttons' => $buttons], [
              'sent_by_ai' => true,
              'flow_id' => $ctx->run->flow_id,
              'flow_run_id' => $ctx->run->id,
              'node_id' => $node->id,
          ]);

          $sent[] = $node->id;
          $context = $ctx->run->context ?? [];
          $context['_sent'] = $sent;
          $ctx->run->update(['context' => $context]);
          return NodeResult::wait(); // dừng chờ postback
      }
  }
  ```
- `app/app/Modules/Messaging/Listeners/AdvanceFlowOnPostback.php`:
  ```php
  <?php
  namespace CMBcoreSeller\Modules\Messaging\Listeners;
  use CMBcoreSeller\Modules\Messaging\Events\PostbackReceived;
  use CMBcoreSeller\Modules\Messaging\Models\Conversation;
  use CMBcoreSeller\Modules\Messaging\Models\FlowRun;
  use CMBcoreSeller\Modules\Messaging\Services\Flows\FlowEngine;
  use CMBcoreSeller\Modules\Messaging\Services\Flows\FlowPostbackPayload;
  use CMBcoreSeller\Modules\Tenancy\Scopes\TenantScope;
  use Illuminate\Contracts\Queue\ShouldQueue;
  /**
   * PostbackReceived → giải mã payload {node_id, handle}; nếu có run waiting đúng
   * node đã gửi nút ⇒ resume theo edge handle. Payload không phải flow ⇒ bỏ qua.
   */
  class AdvanceFlowOnPostback implements ShouldQueue
  {
      public string $queue = 'messaging';
      public function __construct(private FlowEngine $engine) {}
      public function handle(PostbackReceived $event): void
      {
          $decoded = FlowPostbackPayload::decode($event->payload);
          if ($decoded === null) { return; }
          $conv = Conversation::withoutGlobalScope(TenantScope::class)->find($event->conversationId);
          if (! $conv) { return; }
          $run = FlowRun::withoutGlobalScope(TenantScope::class)
              ->where('tenant_id', $conv->tenant_id)
              ->where('conversation_id', $conv->id)
              ->where('status', FlowRun::STATUS_WAITING)
              ->first();
          // Stale guard: chỉ resume nếu run đang chờ đúng node đã gửi bộ nút này.
          if (! $run || (string) $run->current_node_id !== $decoded['node_id']) { return; }
          $this->engine->resume($run, $conv, null, $decoded['handle']);
      }
  }
  ```
